Главная

Перед вами небольшая статья-шпаргалка, которая включает в себя удобные, но не всем знакомые команды Git

1. git init — команда используется для создания нового репозитория или делает репозиторий из существующей директории;


2. git clone — команда создает локальную копию удаленного репозитория в текущей директории;

Примеры:

git clone --depth=1 git://someserver/somerepo "Имя папки" — если надо скопировать только файлы


3. git config — команда используется для настройки значений конфигурации Git на глобальном и локальном уровнях проекта;

Примеры:

git config --local user.name "Name"

git config --local user.email your_email@example.com

git config --global user.name "Name"

git config --global user.email your_email@example.com


4. git remote — команда используется для управления списком удалённых репозиториев;

Примеры:

git remotehttps://github.com/MrKrishnaAgarwal/Git-CheatSheet.git

git remote -v — команда используется для просмотра подключенных удаленных репозиториев;


5. git status — команда используется для просмотра состояния файлов в рабочей директории и индексе: какие файлы изменены, но не добавлены в индекс; какие ожидают коммита в индексе. Вдобавок к этому выводятся подсказки о том, как изменить состояние файлов;


6. git add — команда добавляет содержимое рабочей директории в индекс (staging area) для последующего коммита. По умолчанию git commit использует лишь этот индекс, так что вы можете использовать git add для сборки слепка вашего следующего коммита;


7. git reset — команда используется используется для отмены изменений. Она изменяет указатель HEAD и, опционально, состояние индекса. Также эта команда может изменить файлы в рабочей директории при использовании параметра --hard, что может привести к потере наработок при неправильном использовании, так что убедитесь в серьёзности своих намерений прежде чем использовать его;

Примеры:

git reset --hard HEAD^ — отмена неотправленного коммита

git reset --hard f7880ae2 — отмена отправленного коммита (возврат к состояние коммита f7880ae2)


8. git commit — команда берёт все данные, добавленные в индекс с помощью git add, и сохраняет их слепок во внутренней базе данных, а затем сдвигает указатель текущей ветки на этот слепок

Примеры:

git commit -m “Text message”

git commit -m "added navigation bar"

-m — позволяет указать commit message без обращения к редактору

git commit --amend

--amend — вносит в предыдущий коммит изменения, которые подготовлены к коммиту. Используется и для редактирования предыдущего commit message, если в нём допущены ошибки.


9. git push — команда используется для отправки закоммиченных файлов локального репозитория в удаленный репозиторий в указанной ветке;

Примеры:

git push -f origin

git push -u origin

git push -u origin master

Ключ -u создаёт в удалённом репозитории ветку, соответствующую локальной и связывает их;


10. git fetch — команда используется для связи с удалённым репозиторием и получения из него всех изменений, которых у вас пока нет и сохранения их локально;


11. git pull — команда работает как комбинация команд git fetch и git merge, т.е. Git вначале забирает изменения из указанного удалённого репозитория, а затем пытается слить их с текущей веткой;


12. git branch — команда используется для создания, просмотра переименования и удаления веток;

Примеры:

git branch -a — используется для предварительного просмотра всех веток в удаленном репозитории;

git branch -r — используется для предварительного просмотра всех веток на сервере;


13. git checkout — команда используется для переключения между ветками;

Примеры:

git checkout master или git checkout develop


14. git merge — команда используется для объединения двух веток. Для этого укажите ветку, в которую вы хотите унаследовать изменения из текущей;

Примеры:

git merge develop

git merge — abort — команда используется для отмены слияния;

Если нет ошибок, то слияние будет успешным. Прерывание можно использовать в ситуациях, когда слияние не удалось. В этом случае терминал сообщит о конфликте и предложить его исправить.

git merge -X theirs develop


15. git diff — команда используется для вычисления разницы между любыми двумя Git деревьями. Это может быть разница между вашей рабочей директорией и индексом (собственно git diff), разница между индексом и последним коммитом (git diff --staged), или между любыми двумя коммитами (git diff master branchB);


16. git stash — команда используется для временного сохранения всех незакоммиченных изменений для очистки рабочей директории без необходимости коммитить незавершённую работу в новую ветку;

Примеры:

git stash pop — позволяет применить ранее отложенные изменения.
